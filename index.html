Modbus

Posted on 2026-06-25 Edited on 2026-06-29

Modbus 通讯的分类。

Modbus is a widely used communication protocol in industrial automation, supporting two primary modes:ASCLL and RTU.
Each mode has distinct characteristics, advantages, and limitations, making them suitable for different use cases.

Modbus ASCLL

In ASCLL mode, data is transmitted as human-readable ASCLL characters. Each byte is represented by two ASCLL characters, making it easier to debug and interpret.

Modbus RTU

In RTU mode, data is transmitted in a compact binary format, making it more efficient for high-speed communication.

Modbus RTU 概述。

在实际使用中，Modbus RTU的出场率要远比Modbus ASCLL出场率高地多。Modbus RTU根据通信介质的不同，可以分Modbus RTU,Modbus RTU over TCP 和 Modbus TCP.其中，Modbus RTU和Modbus RTU over TCP传输的报文是一模一样的。只是，传输的介质以及方式有差别。Modbus RTU通过双绞线直连，和485通信非常相似。Modbus RTU over TCP是通过网线传输，通信双方具有主从角色以及从机具有iD。Modbus RTU over TCP与Modbus TCP的通信报文差别是 Modbus TCP通信具有TCP的报文头MBAP头。

Modbus功能码

0x01 读取线圈/离散量输出值

用于读取从设备的线圈或离散量输出的状态，即各DO(Discrete Output,离散输出)的ON/OFF状态。即0或1.
需要注意，在Modbus协议规定的PDU中，规定所哟线圈或寄存器地址从0开始计算。

0x02 读取离散量输入值

用于读取从设备的离散输入，即DI(Discrete Input)的ON/OFF状态。即0或1.
消息帧中指定了需读取的离散输入寄存器起始地址和数目，可读取1-2000个连续的离散量输入状态.

DO，即离散输出可读可写；DI，即离散输入，只可读。

0x03 读取保持寄存器值

用于读取从设备的保持寄存器的内容。不支持广播。
寄存器数量有2个字节构成，最多一次可连续读取125个寄存器值。
需要注意，Modbus的保持寄存器和输入寄存器是以字(Word)为基础单位的(1Word = 2 Bytes)。因此，在读取时需要注意字节序(大小端)问题。

0x04 读取输入寄存器值

用于读取从设备输入寄存器的内容，不支持广播模式。
保持寄存器可读可写，输入寄存器只可读。

0x05 写单个线圈/单个离散输出

用与将单个线圈寄存器设置为ON或OFF,支持广播模式。广播模式对所有从设备生效。

0x06 写单个保持寄存器

用于更新从设备的单个保持寄存器的值，支持广播模式。
保持寄存器以字(Word)为基本单位，写入时需要注意目标数据的字节序问题。

0x08 诊断功能

用于串行链路，主要用于检测主设备和从设备之间的通信故障，或检测从设备的各种内部故障。不支持广播。
查询报文中需要指定从设备地址、功能码(Modbus Command)以及子功能码(Diagnostic Subfunction)。其中，子功能码为2个字节，用于区别各诊断类型。
在正常的响应报文中，从设备将原样回复功能码和子功能码。

功能码 子功能码 描述 说明
08 00(0x00) Return Data Query(Loop-back) 原样返回查询报文
08 01(0x01) ReStart Communication 用于初始化并重启从设备，其中报文字段0x00,0x00表示保持时间记录0xFF,0x00表示清除事件记录
08 03(0x03) Change ASCll Input Delimiter
08 04(0x04) Force Listen Only Mode 强制被寻址的从站设备进入只听模式
使其与网络中的其他设备断开，不返回响应
08 10(0x0A) Clear Counters and Diagnostic Registers 清除计数器和诊断寄存器
08 11(0x0B) Return Bus Message Count 返回总线报文计数值
08 12(0x0C) Return Bus Communication Error Count 返回总线通信 CRC 出错计数
08 13(0x0D) Return Bus Exception Error Count 返回总线异常计数
08 14(0x0E) Return Slave Message Count 返回从站设备接收的报文数量
08 15(0x0F) Return Slave No Response Count 返回从站设备没有返回响应的报文数量
08 16(0x10) Return Slave NAK Count
08 17(0x11) Return Slave Busy Count 返回从站设备响应忙的报文数量
08 18(0x12) Return Bus Character Overrun Count 返回总线字符超限的报文数量
08 19(0x13) Return IOP Overrun Count(884)
08 20(0x14) Clear Overrun Counter and Flag(884)

0x0B 获取通信事件计数器

主要用于获取从设备通信计数器中的状态字和事件计数的值，不支持广播。
可以通过在通信报文之前和之后读取通信事件计数值，来确定从设备是否正常处理报文。
对于正常完成报文处理和传输的场合，事件计数器增加1；而对于异常响应、轮询命令或读事件计数器（即0x0B功能码）的场合，则计数器不变。
通过0x0B诊断功能中的0x01子功能和0x0A子功能，可以复位事件寄存器。

0x0C 获取通信事件记录

用于从从设备获取状态字、事件计数、报文计数以及事件字节字段。
其中状态字和事件计数和功能码0x0B获取的值一致。
报文计数器包含了加电重启、清除计数器之后的报文数量，报文计数与通过0x0B诊断功能中的0x0B子功能码获取的值一致。
事件字节字段包含0-64个字节，定义各种事件。
正常情况下响应报文包括一个2Bytes状态字字段、一个2Bytes事件计数字段、一个2Bytes消息计数字段以及0-64个字节的事件字段。
由于事件字段是变长的，因此增加了一个1Bytes的数据长度字段，以方便读取响应数据。

0x0F 写多个线圈

用于将连续的多个线圈设置为ON或OFF，支持广播模式。
消息帧中指定了需要变更的线圈起始地址和线圈数目。
起始地址由2个字节构成，取值范围为0x0000-0xFFFF。
寄存器数量字段由2个字节构成，取值范围为0x0001到0x07B0。
数据字段中为逻辑1的位对应ON，逻辑0的位对应OFF。

0x10 写多个保持寄存器

用于设置或写入从设备保持寄存器的多个连续的地址块（1-123个寄存器），支持广播模式。
消息帧中需要指定从设备地址以及需要变更的保持寄存器的地址和数量。
起始地址由2个字节构成，取值范围为0x0001到0x007B，（即十进制1-123）。
在实际开发中，该功能码常用于方便用户写入多字节类型的数据，例如浮点数值。需要注意字节序问题。

0x11 报告从站ID

仅适用于串行链路。用于读取从机设备的ID、类型描述、当前状态以及其他信息。不支持广播。
查询报文中没有数据字段。
响应消息的构成由从站设备决定。

Modbus 数据校验

Modbus RTU 和 Modbus ASCll 通信时都会涉及到数据校验。但是两者使用不同的方式进行数据校验。Modbus TCP不需要数据校验。

Modbus RTU over TCP

可以理解为用TCP的方式传输Modbus RTU。TCP的方式就会涉及到IP以及端口。硬件设置好之后，就和TCP无关了。

ASCll 模式 数据校验

使用LRC(Longitudinal Redundancy Check，纵向冗余校验)的计算结果。计算对象不包括起始的冒号（：）和回车换行符号（CRLF）。

RTU 模式 数据校验

使用CRC(Cyclical Redundancy Check，循环冗余校验)的计算结果。计算对象包括校验域之前的所有字节。
